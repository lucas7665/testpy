# 标准化服务网站页面预览

这里包含了网站主要页面的HTML静态预览，可以直接在浏览器中打开查看效果。

## 页面列表

### 1. 首页 (index.html)
- **功能**：网站首页，包含导航栏、Banner、核心功能入口、搜索区、产品亮点、服务介绍等
- **设计特点**：
  - 渐变背景Banner
  - 卡片式功能入口
  - 大型搜索框
  - 响应式布局

### 2. 标准查询结果页 (search.html)
- **功能**：标准查询结果展示页面
- **设计特点**：
  - 左侧筛选面板（标准类型、状态、年份、机构）
  - 右侧结果列表（包含标准号、名称、发布时间等）
  - 分页功能
  - 结果排序

### 3. 标准详情页 (detail.html)
- **功能**：标准详细信息展示页面
- **设计特点**：
  - 标准基本信息展示
  - 标签页切换（摘要、目录、全文、关联标准）
  - 右侧操作面板（快速操作、下载、统计）
  - PDF预览区

### 4. AI工作台 (ai-workspace.html)
- **功能**：AI服务统一入口和工作台
- **设计特点**：
  - 使用统计展示
  - 4个AI功能卡片（评估、编写、审校、问答）
  - 历史记录列表
  - 渐变背景设计

### 5. 标准体系管理页 (system.html)
- **功能**：标准体系的创建和管理
- **设计特点**：
  - 左侧树形结构编辑器
  - 右侧节点编辑表单
  - 标准列表管理
  - 支持多级分类

## 如何使用

1. **直接打开**：在浏览器中直接打开任意HTML文件即可查看
2. **本地服务器**（推荐）：
   ```bash
   # 使用Python
   cd pages
   python -m http.server 8000
   
   # 然后在浏览器访问
   # http://localhost:8000/index.html
   ```

## 注意事项

1. 这些是静态HTML页面，数据都是测试数据
2. 页面间的链接需要根据实际情况调整路径
3. 某些交互功能（如搜索、筛选）在静态页面中不会真正工作
4. 实际开发时需要将样式提取到单独的CSS文件

## 后续开发建议

1. **分离样式**：将CSS提取到独立文件，方便维护
2. **组件化**：将导航栏、底部等公共部分提取为组件
3. **交互实现**：添加JavaScript实现真实的搜索、筛选等功能
4. **数据对接**：连接后端API获取真实数据
5. **框架选择**：可以考虑使用Vue、React等框架进行开发
